# Security Policy

Thank you for helping keep this project secure.

Reporting a vulnerability

- Do not publicly disclose a vulnerability before it is resolved.
- Open a private issue (if possible) or contact the maintainers via GitHub to share sensitive details.
- For critical security issues you can also use GitHub Security Advisories.

What to include in your report

- A short description of the vulnerability
- Steps to reproduce or a proof-of-concept exploit
- Impact and the affected version/branch

Response

Maintainers will aim to respond within a reasonable timeframe and coordinate a fix. After mitigation, a patch will be published and communicated via a release or changelog.

For more information: https://docs.github.com/en/code-security
